Task 1

Есть компания, которая оригинальным способом ведёт свои счета. Каждый раз при записи числа, они перемешивают цифры следующим образом: сначала первую цифру сначала числа, потом первую с конца, потом вторую от начала числа, потом вторую с конца и так до тех пор, пока длина финального числа не будет равна длине начального числа. Напишите функцию solution(myNumber), которая получив целое число, возвращает перевёртыш, например, дано 123456, вернёт 162534. Дано 130, вернёт 103. Число может быть от 0 до 100 000 000. В решении главное правильность, а не скорость выполнения.

Task 2

В index.js файле есть Array объектов unsorted. Написать: 1) функцию, которая будет сортировать элементы сначала по id, потом по profile; 2) функцию, которая будет возвращать массив, в котором каждый элемент также имеет свойство ‘discount’, значение которого рассчитывается, как 5% от amount данного элемента, а свойства profile не будет вообще; 3) функцию, которая будет принимать, как аргументы исходный массив и массив индексов (например, [1, 3, 4]), в результате она будет заменять элементы исходного массива на объект типа { removed: true }.

Task 3

Есть форум с лимитом в K символов для 1-го комментария. Ваша задача - разработать функцию, чтобы обрезать слишком длинные комменты. Вам даётся строка из английских букв и пробелов. Нужно обрезать некое количество слов с конца строки, придерживаясь таких правил: * нельзя отрезать кусок слова; * строка-результат не должна заканчиваться на пробел; * строка-результат не должна быть длиннее, чем лимит К, который передаётся в фукцию, как аргумент; * строка-результат должна быть максимально длинной. Это значит, что, в некоторых случаях, функция должна отрезать всю строку, выдавая пустую строку. Например, дано "Andersen We Test Programmers" c K = 14, вернёт "Andersen We". Имейте в виду, что * результат "Andersen We Test Pr" - неправильно, так как разрезано слово; * результат "Andersen We " - неправильно, так как заканчивается на пробел; * результат "Andersen We Test Programmers" - неправильно, так как превосходит лимит; * результат "Andersen" - неправильно, так как короче, чем правильный результат;
